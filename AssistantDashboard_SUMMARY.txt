================================================================================
                    ACTION CALLBACKS (PCF OUTPUTS)
================================================================================

The following callbacks are emitted from components → index.ts → Canvas app:

  onSelectCard(cardId: string)
    - Emitted by: CardItem, BriefingCard
    - Output: selectedCardId
    - Canvas app uses: Read card details

  onSendDraft(cardId: string, finalText: string, editDistanceRatio: number)
    - Emitted by: CardDetail
    - Output: sendDraftAction = { cardId, finalText, editDistanceRatio }
    - Canvas app uses: Send email/teams message, track edit distance
    - Edit distance: 0 = sent as-is, >0 = edited

  onCopyDraft(cardId: string)
    - Emitted by: CardDetail
    - Output: copyDraftAction
    - Canvas app uses: Copy to clipboard (user interaction)

  onDismissCard(cardId: string)
    - Emitted by: CardItem (context menu), CardDetail, BriefingCard
    - Output: dismissCardAction
    - Canvas app uses: Update cr_cardoutcome = "DISMISSED"
    - Retry logic: index.ts retries dismissals every 5 seconds for 3 attempts

  onJumpToCard(cardId: string)
    - Emitted by: BriefingCard action items
    - Output: jumpToCardAction
    - Canvas app uses: Navigate to related card (Sprint 2)

  onExecuteCommand(command: string, currentCardId: string | null)
    - Emitted by: CommandBar
    - Output: commandAction = { command, currentCardId }
    - Canvas app uses: Call PowerAutomate orchestrator (Sprint 3)

  onSaveDraft(cardId: string, editedText: string)
    - Emitted by: CardDetail
    - Output: saveDraftAction = { cardId, editedText }
    - Canvas app uses: Persist draft edits to cr_humanizeddraft (Phase 18)

  onUpdateSchedule(config: BriefingScheduleConfig)
    - Emitted by: BriefingCard
    - Output: updateScheduleAction = JSON.stringify(config)
    - Canvas app uses: Set briefing schedule (Sprint 2)

When porting, you'll replace PCF output mechanism with:
  - Event emitters / callbacks
  - Context API
  - State management (Redux, Zustand, etc.)
  - API calls to your backend
